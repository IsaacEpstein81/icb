Sprint Objective

Deploy the first launch-ready version of ICB’s professional learning platform, with the branded LMS, embedded AI copilot, and CPSW training path ready for real use.

Why CPSW First

CPSW is the most urgent near-term opportunity, so the platform and module can be built in parallel without waiting for a broader multi-training roadmap to be fully defined.

Commercial Path

The first module is not a one-off build. It creates a reusable platform that can later support Ethics, Crisis, Supervisor, and either a $2,000 per month per course model or a revenue-share model.

Engagement Phases

Phase 1 launches the platform. Phase 2 expands the training catalog.

The first phase is a Professional Learning Platform Activation Sprint focused on launch readiness. The second phase adds future trainings once the platform and CPSW module are live.

Phase 1

Professional Learning Platform Activation Sprint

Objective: deploy the first operational version of ICB’s learner-facing LMS and AI support environment, with the initial CPSW module built and launch-ready.

  • ICB branding and LMS environment setup
  • Course and lesson structure configuration
  • AI copilot setup and placement within the learner experience
  • CPSW module development using available source materials and related reference materials where applicable
  • Exercises, assessments, and learner flow configuration
  • Administrative review, testing, QA, and launch preparation

Phase 2

Additional Training Expansion

After launch of the initial LMS and CPSW module, OpenRecovery and ICB can expand the platform with additional trainings and credential-related modules.

  • Add trainings such as Ethics, Crisis, Supervisor, or other credential-linked modules
  • Implement directly from existing source materials where available
  • Add curriculum-development support where needed
  • Scope future work based on training length, material quality, and review requirements

Implementation Considerations

Protect speed by aligning materials, assessments, and review cadence early.

A few practical decisions determine how quickly CPSW and future trainings can move from concept into a live learner experience.

Considerations

  • More usable source materials mean faster module development
  • Assessments should stay aligned to the final training content
  • Videos and rich media can extend timing if treated as separate production tracks
  • Weekly feedback and batched approvals help keep everything moving in parallel

Exclusions

  • Unlimited net-new curriculum writing across multiple trainings
  • Multiple fully built modules beyond the agreed first module
  • Advanced custom integrations or enterprise reporting outside the agreed LMS scope
  • Ongoing maintenance for future content revisions unless separately scoped
